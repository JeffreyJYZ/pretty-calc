# Pretty Calc

A pastel-themed scientific + basic calculator for desktop and mobile. Built as a Tauri 2 app with a hybrid TS / Rust engine: TypeScript drives the live preview on every keystroke, Rust evaluates committed results via a single IPC call for exactness and speed.

Cross-platform: **macOS**, **Windows**, **Linux**, **iOS**, **Android**.

## Features

- Basic + Scientific modes (toggle in header)
- Trig: sin/cos/tan, arc (asin/acos/atan), hyperbolic (sinh/cosh/tanh), arc-hyperbolic
- Logs: ln, log10, log2; Powers: x², x³, x^y, √, ∛, 1/x; Factorial, Percent, Modulo, abs, floor/ceil/round/sign
- Implicit multiplication (`2π`, `3(4+5)`), unary minus, parentheses
- Constants: π, e, Ans (last result)
- 2nd shift + HYP keys (Casio-style)
- Memory: MC / MR / M+ / M-
- History panel with reuse (insert expr or result)
- DEG / RAD angle modes
- Persistent state across restarts (history, memory, Ans, angle/mode preference) via `localStorage`
- Full keyboard support (numbers, operators, Enter/=, Backspace, Esc, parens)
- Accessible: ARIA labels, focus-visible, reduced-motion, safe-area insets for notched phones
- Responsive: 4-col keypad on phones, 5-col scientific grid on wider screens, light/dark pastel themes
- Custom SVG logo + platform-perfect icons (auto-generated by `tauri icon`)

## Tech stack

- **Tauri 2** (Rust shell, IPC commands) — `src-tauri/`
- **React 19** + **TypeScript 7** — `src/`
- **Vite 8** + **@vitejs/plugin-react 6** (rolldown-mode)
- **Biome 2** (lint + format, single tool)
- **Bun** (runtime, package manager, script runner)
- Pure-TS live-preview engine (`src/lib/calc.ts`) + pure-Rust committed-eval engine (`src-tauri/src/calc.rs`)
- No external math dependency. Both engines are self-contained tokenizer → shunting-yard (RPN) → evaluator implementations.

## Getting started

```bash
bun install
```

### Develop (desktop)

```bash
bun tauri dev
```

Vite runs on `http://localhost:1420` (strict port), Tauri opens a native window pointing at it. Rust changes hot-reload via cargo, frontend via Vite HMR.

### Build production desktop bundle

```bash
bun tauri build
```

Outputs installers + bundles in `src-tauri/target/release/bundle/` (`.app`, `.dmg`, `.msi`, `.deb`, `.AppImage`, etc. depending on host OS).

### Run the web frontend only (no Tauri)

```bash
bun run dev
```

Pure browser mode falls back to the TS engine for both preview and committed `=` (since the Rust `calc_eval` IPC command is unavailable).

## Mobile

iOS and Android require Tauri's mobile subcommand setup. The repo already has Rust targets installed and the iOS Xcode project generated at `src-tauri/gen/apple/`.

### iOS

Prereqs: Xcode, `cargo install tauri-cli --version "^2"` (or use `bun tauri ios`).

```bash
bun tauri ios init      # already done; safe to re-run
bun tauri ios dev       # run on simulator or connected device
bun tauri ios build     # produce .ipa
```

### Android

Prereqs: Android Studio (or standalone SDK + NDK + cmdline-tools). Set `ANDROID_HOME` to your SDK root (Tauri auto-detects Android Studio's default location on macOS).

Install NDK + cmdline-tools via Android Studio's SDK Manager (Tauri needs `ndk;27.0.12077973` or newer + platform-tools + platforms;android-34).

```bash
bun tauri android init
bun tauri android dev
bun tauri android build
```

If `bun tauri android init` fails with "Skipping Android Studio command line tools installation", finish the manual setup described at https://tauri.app/start/prerequisites/#android (install cmdline-tools via SDK Manager → SDK Tools tab → "Android SDK Command-line Tools (latest)").

The logo source is `public/logo.svg`. After editing, regenerate all platform icons (PNG sizes, `.icns`, `.ico`, iOS `AppIcon` set, Android `mipmap` set):

```bash
bun tauri icon public/logo.svg
```

This overwrites `src-tauri/icons/`. Commit the result.

## Engine design

Two independent engines implement the same grammar:

1. **TS engine** (`src/lib/calc.ts`) — synchronous, no IPC. Used for the live preview that re-evaluates as you type. Returns `null` on parse/eval errors silently.
2. **Rust engine** (`src-tauri/src/calc.rs`) — invoked via the `calc_eval` Tauri command on `=` press. Returns either a formatted result string or `__err:<message>` on failure.

Both tokenize → shunting-yard → RPN eval on f64. Same precedence/associativity rules, same function table, same angle-mode handling. The TS version exists because Tauri IPC per keystroke would dominate cost for the small expressions typical of calculator input; the Rust version is preferred for committed results because it avoids double-parsing and gives exact f64 throughput with no GC pauses.

Hybrid wiring lives in `src/components/Calculator.tsx`: runtime detection via `window.__TAURI_INTERNALS__` decides which engine serves `=`. Pure-web mode uses TS for both.
